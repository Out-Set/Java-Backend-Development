This is not a good approach: bcoz
=================================
 -> Each time there is a new listener, you need to inject it into publisher class.
 -> And then call them.

Better implementation:
======================

Config class:
-------------
@Configuration
@ComponentScan("com.savan")
public class AppConfig {

}

Main class:
-----------
public class Main {
    public static void main(String[] args) {
        ApplicationContext context = new AnnotationConfigApplicationContext(AppConfig.class);
        ZeeCafePublisher bean = context.getBean("zeeCafePublisher", ZeeCafePublisher.class);
        bean.streamBigBangTheory("EP - 004");
    }
}

Event:
------
public class BigBangTheoryEvent extends ApplicationEvent {

    private String episodeNo;

    public BigBangTheoryEvent(Object source) {
        super(source);
    }

    public BigBangTheoryEvent(Object source, String episodeNo) {
        super(source);
        this.episodeNo = episodeNo;
    }

    public String getEpisodeNo() {
        return episodeNo;
    }
}

Publisher:
----------
@Component
public class ZeeCafePublisher {

    @Autowired
    private ApplicationEventPublisher applicationEventPublisher;

    public void streamBigBangTheory(String episodeNo) {
        System.out.println("Zee Cafe : Starting BBT: "+episodeNo);

        applicationEventPublisher.publishEvent(new BigBangTheoryEvent(this, episodeNo));
    }
}

Listeners:
----------
@Component
public class SavanListener implements ApplicationListener<BigBangTheoryEvent> {

    public void watchBigBang(String episodeNo) {
        System.out.println("Savan started watching BBT, episode-no: "+episodeNo);
    }

    @Override
    public void onApplicationEvent(BigBangTheoryEvent event) {
        watchBigBang(event.getEpisodeNo());
    }
}

@Component
public class AmanListener implements ApplicationListener<BigBangTheoryEvent> {

    public void watchBigBang(String episodeNo) {
        System.out.println("Aman started watching BBT, episode-no: "+episodeNo);
    }

    @Override
    public void onApplicationEvent(BigBangTheoryEvent event) {
        watchBigBang(event.getEpisodeNo());
    }
}

o/p on main() run:
------------------
Zee Cafe : Starting BBT: EP - 004
Aman started watching BBT, episode-no: EP - 004
Savan started watching BBT, episode-no: EP - 004

Event:
------
public class BigBangTheoryEvent {

    private String episodeNo;

    public BigBangTheoryEvent(String episodeNo) {
        this.episodeNo = episodeNo;
    }

    public String getEpisodeNo() {
        return episodeNo;
    }
}

public class ComedyCircusEvent {

    private String episodeNo;

    public ComedyCircusEvent(String episodeNo) {
        this.episodeNo = episodeNo;
    }

    public String getEpisodeNo() {
        return episodeNo;
    }
}

Publisher:
----------
@Component
public class ZeeCafePublisher {

    @Autowired
    private ApplicationEventPublisher applicationEventPublisher;

    public void streamBigBangTheory(String episodeNo) {
        System.out.println("Zee Cafe : Starting BBT: "+episodeNo);
        applicationEventPublisher.publishEvent(new BigBangTheoryEvent(episodeNo));
    }

    public void streamComedyCircus(String episodeNo) {
        System.out.println("Zee Cafe : Starting Comedy Circus: "+episodeNo);
        applicationEventPublisher.publishEvent(new ComedyCircusEvent(episodeNo));
    }
}

public class AmanListener {

        watchBigBang(event.getEpisodeNo());

}

Note: 
=====
 -> All the listeners are executed synchronously.
 -> If one listener is blocked then the others has to wait till it completes its execution in order to start their work.

example:
--------
@EventListener
public void justinListenToEvent(BigBangTheoryEvent event) {
    try {
         Thread.sleep(5000);
    } catch (InterruptedException e) {
         throw new RuntimeException(e);
    }
    watchComedyCircus(event.getEpisodeNo());
}
